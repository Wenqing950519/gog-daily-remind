# 模組：Google 行事曆每日簡報

## 概述

每天早上自動掃描 Google Calendar，將今日行程整理成結構化訊息發送到 Telegram。

> ⚠️ 使用本模組前請先完成 `setup/gog-setup.md` 的安裝與授權流程，
> 授權時 `--services` 需包含 `calendar`。

---

## 模組變數

| 變數 | 說明 | 預設值 |
|------|------|--------|
| `ENABLE_CALENDAR` | 是否啟用本模組 | `false` |
| `CALENDAR_CRON` | 觸發時間（UTC Cron） | `0 23 * * *` |
| `GOG_TIMEZONE` | 用戶所在時區（用於判斷「今天」） | 必填 |

---

## 觸發條件

### 定時自動觸發
- **預設時間：** 每天早上 07:00（依 `GOG_TIMEZONE` 換算）
- **Cron 範例（Asia/Taipei，UTC+8）：** `0 23 * * *`（台北 07:00 = UTC 23:00）
- 可透過 `CALENDAR_CRON` 調整觸發時間

### 手動關鍵字觸發
- 「今天有什麼行程」
- 「查行事曆」
- 「今天的待辦」
- 「calendar brief」
- 「行程提醒」

---

## 執行步驟

### Step 1：取得今日行程

```bash
gog calendar events --today --json
```

### Step 2：解析與分類事件

從 JSON 回應的 `events` 陣列中，每筆事件擷取以下欄位：

| 欄位 | 說明 |
|------|------|
| `summary` | 事件名稱 |
| `start.dateTime` / `start.date` | 開始時間（有時間 vs 全天）|
| `end.dateTime` / `end.date` | 結束時間 |
| `startLocal` / `endLocal` | 本地時間字串（優先使用）|
| `description` | 備註說明（有值才顯示）|
| `location` | 地點（有值才顯示）|

分類規則：
- **有時間的行程** — 有 `start.dateTime`，依 `startLocal` 排序，顯示於上方
- **全天事項** — 只有 `start.date`，顯示於下方，以分隔線區隔

### Step 3：格式化 Telegram 訊息

```
🗓 [MM/DD 週X]

• [HH:MM] [事件名稱]
  📍 [地點]
  📝 [備註]

• [HH:MM–HH:MM] [事件名稱]

────────────────
• [全天事項名稱]
• [全天事項名稱]

共 [N] 個行程，祝今天順利 ✨
```

範例輸出：
```
🗓 03/12 週三

• 09:30 晨會
  📍 Google Meet
  📝 Q2 規劃討論

• 14:00–15:30 與設計師對稿

────────────────
• 繳房租截止日
• 老媽生日

共 4 個行程，祝今天順利 ✨
```

若今天沒有任何行程：
```
🗓 [MM/DD 週X]

今天沒有任何行程，是個自由的好日子 🌿
```

> 空白欄位一律省略（無地點就不顯示 📍 那行）
> 無全天事項時省略分隔線區塊

### Step 4：發送到 Telegram

使用標準 Telegram 通知方式，將格式化訊息發送到指定群組或對話。

---

## 錯誤處理

| 情況 | 處理方式 |
|------|----------|
| `gog` 指令失敗 / 授權錯誤 | 發送：`⚠️ 無法取得行事曆，請檢查 Google 授權狀態。` |
| JSON 解析失敗 | 發送：`⚠️ 行事曆資料解析失敗，請稍後再試。` |
| 沒有任何事件 | 發送「今天沒有行程」版本訊息（見 Step 3）|

---

## 注意事項

- 「今天」的判斷時區以 `GOG_TIMEZONE` 為準
- Telegram 訊息不得包含原始 JSON 或指令輸出
